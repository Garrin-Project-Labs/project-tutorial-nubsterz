# Domain docs

Layout: single-context.

- Read `CONTEXT.md` first for project language.
- Use `docs/adr/` only for decisions that are hard to reverse, surprising without context, and based on a real trade-off.
- Do not create heavy docs for fun projects unless they clarify real confusion.
- Keep user-facing language simple; hide PRD/issues/TDD terminology unless a power user asks for it.
