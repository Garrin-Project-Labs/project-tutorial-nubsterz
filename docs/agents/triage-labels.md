# Triage labels

This project uses local task state instead of issue labels by default.

Canonical state mapping:

- `needs-triage` → needs a maintainer/friend to decide what this means
- `needs-info` → waiting on a person to answer a concrete question
- `ready-for-agent` → ready for an AFK agent to work on
- `ready-for-human` → needs human taste, design, access, or judgment
- `wontfix` → intentionally not doing this

In Discord, keep the wording friendly: "needs a decision", "waiting on an answer", "ready for bot help", "needs a human", "not for now".
